# Implementation Plan: Extent Manager Benchmark Application

**Branch**: `002-extent-benchmark` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-extent-benchmark/spec.md`

## Summary

Build a standalone CLI benchmark application (`apps/extent-benchmark/`) that measures latency and throughput of the ExtentManagerComponentV1's three core operations: create_extent, lookup_extent, and remove_extent. The app wires the full SPDK component stack (Logger → SPDKEnv → BlockDeviceSpdkNvme → ExtentManager), supports multi-threaded operation via `--threads N`, and reports per-phase latency percentiles (min, p50, p99, max) and aggregate throughput (ops/sec). Follows the established `iops-benchmark` wiring pattern.

## Key Implementation Details

### Component Wiring (main.rs)

Reference: `apps/iops-benchmark/src/main.rs`

```
LoggerComponent::new()
SPDKEnvComponent::new_default()
BlockDeviceSpdkNvmeComponentV1::new_default()
ExtentManagerComponentV1::new_default()

bind: logger → spdk_env.logger
bind: logger → block_dev.logger
bind: spdk_env → block_dev.spdk_env
bind: block_dev → extent_mgr.block_device
bind: logger → extent_mgr.logger

ISPDKEnv::init() → devices() → select device by PCI address
IBlockDeviceAdmin::set_pci_address() → initialize()
IExtentManagerAdmin::set_dma_alloc() → initialize(total_size, slab_size, ns_id)
```

### Worker Thread Design (worker.rs)

- Each thread receives `Arc<ExtentManagerComponentV1>` (component is `Send + Sync`)
- Key ranges: thread `t` with `count_per_thread` ops uses keys `[t * count_per_thread .. (t+1) * count_per_thread)`
- `Barrier` synchronization: all threads wait at barrier before each phase starts
- Per-operation: `let start = Instant::now(); op(); latencies.push(start.elapsed());`

### Statistics (stats.rs)

- Sort `Vec<Duration>`, compute percentiles by index: `p50 = samples[len/2]`, `p99 = samples[len*99/100]`
- Mean = sum / count
- Return `LatencyStats { count, min, max, mean, p50, p99 }`

### DMA Allocator

- Use `DmaBuffer::new(size, align, numa_node)` from the SPDK environment, not `heap_dma_alloc`
- Obtain NUMA node from `IBlockDevice::numa_node()`
- Set via `IExtentManagerAdmin::set_dma_alloc()` with a closure that calls `DmaBuffer::new`

### Error Handling

- SPDK init failures → print error, exit(2)
- Invalid CLI args → clap handles, exit(1)
- Mid-phase errors → log error count, report partial results, continue to next phase
