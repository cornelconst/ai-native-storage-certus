# Data Model: SPDK NVMe Block Device Component

## Entities

### BlockDeviceSpdkNvmeComponent

The top-level component struct, generated by `define_component!`.

| Field | Type | Description |
|-------|------|-------------|
| logger | `Receptacle<dyn ILogger>` | Debug logging receptacle (auto-generated) |
| spdk_env | `Receptacle<dyn ISPDKEnv>` | SPDK environment receptacle (auto-generated) |
| controller | `RwLock<Option<NvmeController>>` | Attached NVMe controller state |
| actor_handle | `Mutex<Option<ActorHandle<ControlMessage>>>` | Handle to the running actor thread |
| pci_address | `PciAddress` | PCI BDF of the NVMe controller to attach |

### NvmeController

Internal representation of an attached NVMe controller.

| Field | Type | Description |
|-------|------|-------------|
| ctrlr_ptr | `*mut spdk_nvme_ctrlr` | Raw SPDK controller pointer |
| qpairs | `Vec<QueuePair>` | Pool of IO queue pairs with varying depths |
| namespaces | `Vec<NvmeNamespaceInfo>` | Discovered namespaces |
| numa_node | `i32` | NUMA node of the controller device |
| version | `NvmeVersion` | NVMe specification version |
| max_transfer_size | `u32` | Maximum data transfer size in bytes |
| max_queue_depth | `u32` | Maximum queue depth |
| num_io_queues | `u32` | Number of IO queues |

### QueuePair

An SPDK NVMe IO queue pair.

| Field | Type | Description |
|-------|------|-------------|
| qpair_ptr | `*mut spdk_nvme_qpair` | Raw SPDK queue pair pointer |
| depth | `u32` | Queue depth |
| in_flight | `u32` | Number of currently in-flight operations |

### NvmeNamespaceInfo

| Field | Type | Description |
|-------|------|-------------|
| ns_id | `u32` | NVMe namespace identifier |
| ns_ptr | `*mut spdk_nvme_ns` | Raw SPDK namespace pointer |
| num_sectors | `u64` | Total sector count |
| sector_size | `u32` | Sector size in bytes |

### ClientSession

Per-client state maintained by the actor handler.

| Field | Type | Description |
|-------|------|-------------|
| id | `u64` | Unique client session identifier |
| ingress_rx | `Receiver<Command>` | Receiver end of client's ingress SPSC channel |
| callback_tx | `Sender<Completion>` | Sender end of client's callback SPSC channel |
| pending_ops | `HashMap<u64, PendingOp>` | In-flight async operations for this client |

### PendingOp

Tracks a single in-flight asynchronous IO operation.

| Field | Type | Description |
|-------|------|-------------|
| handle | `u64` | Component-assigned unique operation handle |
| deadline | `Instant` | Timeout deadline |
| qpair_idx | `usize` | Index of the queue pair used |

### Command (Enum)

Messages sent by clients on the ingress channel.

| Variant | Fields | Description |
|---------|--------|-------------|
| ReadSync | `{ ns_id: u32, lba: u64, buf: Arc<Mutex<DmaBuffer>> }` | Synchronous read |
| WriteSync | `{ ns_id: u32, lba: u64, buf: Arc<DmaBuffer> }` | Synchronous write |
| ReadAsync | `{ ns_id: u32, lba: u64, buf: Arc<Mutex<DmaBuffer>>, timeout_ms: u64 }` | Async read |
| WriteAsync | `{ ns_id: u32, lba: u64, buf: Arc<DmaBuffer>, timeout_ms: u64 }` | Async write |
| WriteZeros | `{ ns_id: u32, lba: u64, num_blocks: u32 }` | Write zeros |
| BatchSubmit | `{ ops: Vec<Command> }` | Batch of operations |
| AbortOp | `{ handle: u64 }` | Abort an async operation |
| NsProbe | | Probe namespaces |
| NsCreate | `{ size_sectors: u64 }` | Create namespace |
| NsFormat | `{ ns_id: u32 }` | Format namespace |
| NsDelete | `{ ns_id: u32 }` | Delete namespace |
| ControllerReset | | Hardware reset |

### Completion (Enum)

Messages sent by the actor on the callback channel.

| Variant | Fields | Description |
|---------|--------|-------------|
| ReadDone | `{ handle: u64, result: Result<(), BlockDeviceError> }` | Read completed |
| WriteDone | `{ handle: u64, result: Result<(), BlockDeviceError> }` | Write completed |
| WriteZerosDone | `{ handle: u64, result: Result<(), BlockDeviceError> }` | Write zeros completed |
| AbortAck | `{ handle: u64 }` | Abort acknowledged |
| Timeout | `{ handle: u64 }` | Operation timed out |
| NsProbeResult | `{ namespaces: Vec<NvmeNamespaceInfo> }` | Probe result |
| NsCreated | `{ ns_id: u32 }` | Namespace created |
| NsFormatted | `{ ns_id: u32 }` | Namespace formatted |
| NsDeleted | `{ ns_id: u32 }` | Namespace deleted |
| ResetDone | `{ result: Result<(), BlockDeviceError> }` | Reset completed |
| Error | `{ handle: Option<u64>, error: BlockDeviceError }` | General error |

### ControlMessage (Enum)

Messages on the actor's main MPSC channel for lifecycle/control.

| Variant | Fields | Description |
|---------|--------|-------------|
| ConnectClient | `{ session: ClientSession }` | Register a new client |
| DisconnectClient | `{ client_id: u64 }` | Remove a client |

### TelemetryStats (feature-gated)

| Field | Type | Description |
|-------|------|-------------|
| total_ops | `AtomicU64` | Total completed operations |
| min_latency_ns | `AtomicU64` | Minimum IO latency in nanoseconds |
| max_latency_ns | `AtomicU64` | Maximum IO latency in nanoseconds |
| sum_latency_ns | `AtomicU64` | Sum of all latencies (for mean calculation) |
| total_bytes | `AtomicU64` | Total bytes transferred (for throughput) |
| start_time | `Instant` | When telemetry collection started |

## State Transitions

### Component Lifecycle

```
Created -> Opened (open()) -> Active (actor running) -> Closed (close())
                                  |
                                  v
                              Reset (controller_reset()) -> Active
```

### Client Lifecycle

```
Connected (ConnectClient) -> Active (sending commands) -> Disconnected (channel dropped)
```

### Async Operation Lifecycle

```
Submitted -> Pending (in qpair) -> Completed (callback sent)
                |                        |
                v                        v
            Timed Out (error callback)  Aborted (abort ack callback)
```
