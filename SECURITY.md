# Security Policy

We take the security of this project seriously. If you discover a security vulnerability, please report it so we can address it promptly.

## Reporting a Vulnerability

- Preferred: Open a private security issue on GitHub (choose "Private" when creating an issue) addressed to the maintainers.
- Alternatively, send an email to the maintainers at ai-native-storage@ibm.com with the subject line "Security vulnerability report".

When reporting a vulnerability, please include:

- A clear description of the vulnerability and its impact.
- Step-by-step reproduction instructions or a minimal proof-of-concept (PoC).
- The versions of the project and any relevant dependencies.
- Any logs, stack traces, or configuration details needed to reproduce the issue.

Do not publicly disclose a vulnerability before we have had a chance to address it. Coordinated disclosure helps protect users and allows us to prepare and release a fix.

## Response Process

We will acknowledge reports within 72 hours and provide a high-level timeline for investigation and remediation. Critical vulnerabilities will be prioritized.

After a fix is ready, we will coordinate a disclosure timeline and publish a security advisory or release notes that describe the issue, impact, and mitigation steps.

## Supported Versions

We aim to fix security issues in the currently maintained release series. If your report affects an older unsupported version, we will evaluate and provide guidance where possible.

## Credit

If you would like to be credited for reporting a vulnerability, let us know in your report. We will respect requests for anonymity.

## Third-Party Dependencies

This project depends on third-party libraries. If a vulnerability is in a dependency, we will coordinate with the dependency maintainers and update our dependencies as needed.

## Legal

Please only access systems and data for which you have permission while testing. We expect good-faith security research; we will not take legal action against researchers who follow these guidelines.
